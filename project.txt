Project Overview
The goal is to build a high-precision system tailored for the US Tax & Legal domain. The system should
allow legal professionals to query a knowledge base of legal documents and receive summarized
responses with accurate legal citations.
Milestone 1: Data Ingestion & Pre-processing
Objective: Build a pipeline to process legal-domain documents.
Dataset:
• Acts
• Court Judgments
• POV (Point of View) documents
• Tax Documents
Task:
Implement a parser capable of extracting text while preserving page index for downstream citations.
Milestone 2: Search Architecture & Indexing
Objective: Implement a Hybrid Search architecture.
Requirements:
• Vector Search
• Keyword Search (ELK Stack or equivalent)
• Hybrid Search
Advanced Task (Optional):
• Implement Graph RAG by mapping relationships between legal documents.
Knowledge Standardization:
Use a consistent knowledge structure suitable for LLM retrieval.
Milestone 3: Feature Development
Objective: Develop the core user experience.
Features:
• Natural language Q&A
• Summarization of legal rulings
Verification:
• Every response must include the source document.
• Include the relevant page number/reference wherever applicable.
Milestone 4: Quality Assurance & Evaluation
Objective: Validate the system using a self-created Golden Set.
Task:
Create a Golden Set (Excel/CSV or JSON) containing:
• Sample Query
• Ground Truth Answer
• Source Document
• Relevant Page Number(s), wherever applicable.
Run your system against the Golden Set and measure:
• Retrieval Accuracy
• Faithfulness (absence of hallucinations).
Deliverables
• Architecture Diagram (PDF → Parsing → Hybrid Search → LLM)
• Working Demo (UI or API)
• GitHub Repository
• Deployed Application
• Golden Set (Excel/CSV or JSON)
• Evaluation Report
• Short document describing your approach
